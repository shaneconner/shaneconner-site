The Problem

Most task trackers are isolated checkboxes

Traditional to-do systems treat every task as independent. Check a box and nothing else updates. No relationship awareness. No adaptive cadence. No memory of context beyond completion history.

This project flips that model: every log hits a shared hierarchy, and scheduling continuously adapts from actual behavior. Exercise is the easiest place to visualize the hierarchy, but the same engine powers household, hygiene, wellness, and admin routines.

The Insight

One log can update an entire graph

When you log one leaf task, the system recursively updates every parent it belongs to. That means one click can update specific leaves, intermediate groups, and broad rollups in a single pass.

The visualization below uses exercise because it has dense multi-tag structure (movement, muscles, equipment, energy, objective, type), making the propagation effect obvious.

In Practice

Same data, multiple interfaces

You can use a table UI, a desktop coding workflow, or mobile chat. All three write to the same underlying task graph and frequency model.

The mobile workflow is especially useful when away from a computer, including building climbing workouts on the go with climbing-specific logging guidance.

Evolution

From spreadsheet tracking to agentic execution

The system has evolved in three phases: spreadsheet logging, programmatic scheduling, and now MCP-powered agents. Each phase reduced manual data entry while increasing decision quality.

With MCP, the assistant can call real task tools directly: log chores, inspect urgency, search semantically, and build context-aware plans. The result is a practical Jarvis-style loop: less typing, better planning, and continuously improving personal context.

The Mechanism

Frequency evolves from behavior

Each task starts from a baseline cadence. Then the system adapts using completion timing.

Completed early: interval shrinks (more frequent). Completed late: interval expands (less frequent). The default multiplier is 1.382, creating stable two-way adjustment.

Over time, this produces personalized scheduling without manual rule tuning.

Results

Better structure makes prediction easier

Once logs are structured through hierarchy and adaptive cadence, downstream models get cleaner signals. Sequence and context become easier to learn, enabling useful next-task prediction and recommendation behaviors.

It is a good example of practical AI architecture: LLMs orchestrate, specialized tools execute, and structured data does the heavy lifting.
